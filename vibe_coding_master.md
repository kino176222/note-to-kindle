### 3.1 Global Ruleとは何か（AIの憲法）

![](images/external_image_12.jpg)

これがAntigravityを使いこなすうえで、最重要ポイントです。

まず大事な前提から。  
これから設定する「 [GEMINI.md](http://gemini.md/) （Global Rule）」は、  
AIがこのプロジェクト内で必ず守る“憲法”です。

ここに書かれたルールは、  
AIの気分やノリよりも、常に優先されます。

一度ここに定めておくことで、  
AIは毎回それを前提に考え、行動するようになります。

だからこそ、「最初にこの憲法を入れる」ことがめちゃくちゃ重要です。

  

  

  

  

### 3.2 GEMINI.md を設定する（1分で終わります）

<strong>【コピペ用プロンプト (Global Rule)】</strong>

「暴走させず、壊さず、話が通じる相棒」にするための設定です。

-勝手に実装を進める  
-英語で返してくる

最初にこの「憲法」を入れておくことで、  
AIは必ず確認し、段階的に、安全に開発を進めるようになります。

### 3.3 Node.js の役割（なぜ必要か）

![](images/external_image_14.jpg)

次に、PCに「魔法（プログラム）」を使えるようにする儀式を行います。  
このために必要なのが<strong>「Node.js（ノード・ジェイエス）」</strong>です。  

難しく考えなくて大丈夫です。<strong>「これがないと、そもそも始まらないやつ」</strong>だと思っておいてもらえればOKです。  
実は、私たちが使っている<strong>JavaScript</strong>は、もともと  
<strong>Webブラウザの中で動くために作られた言語</strong>でした。

つまり以前は、

- Chrome や Safari の中では動く

- でも PCそのものを動かすことはできない

──そんな制限があったんです。

そこで登場したのが<strong>Node.js（ノード・ジェイエス）</strong>です。  
Node.js は、  
<strong>JavaScript をブラウザの外（PCやサーバー上）でも動かせるようにする実行環境</strong>。

これを入れることで JavaScript は、

- Webサイトの見た目を動かすだけでなく

- ファイル操作やサーバー処理など

- PCや裏側の処理も担当できる

ようになります。

つまり Node.js は、  
<strong>JavaScript の「活動範囲」を一気に広げる仕組み</strong>です。  
そして、Node.js を入れる最大のメリットが  
<strong>npm（エヌピーエム）</strong>という仕組みを使えるようになること。

  
npm とは、

- 世界中の開発者が作った便利なプログラム部品を

- コマンド一発で追加できる

<strong>JavaScript専用の部品倉庫</strong>です。

ログイン機能、デザイン、アニメーション、開発ツール……  
多くのものを<strong>ゼロから作る必要はありません</strong>。

Node.js を入れるということは、<strong>JavaScriptで「アプリ開発ができる環境」を手に入れる</strong>という意味なんです。

### 7.4 プッシュとプル

ここも一気に片付けましょう。

プッシュ（push）  
→<strong>今のセーブデータをクラウドに預ける</strong>

- PC内の最新コードを

- GitHubのリポジトリにアップロードする

＝「今の状態をセーブしました！」

  
プル（pull）  
→<strong>クラウドにあるセーブデータを持ってくる</strong>

- 別のPCで作業するとき

- 過去の状態に戻したいとき

＝「セーブデータをロードしました！」

  
コミット（commit）は何？  
→<strong>セーブポイントを作ること</strong>

- 「ここまでOK！」という区切りを記録

- プッシュする前の下準備

  
なので関係はこうです👇

-<strong>コミット</strong>：セーブポイント作成

-<strong>プッシュ</strong>：セーブデータをクラウドに送信

-<strong>プル</strong>：セーブデータをクラウドから取得

  

  

<strong>難しく考えなくてOKです</strong>
